Portfolio (Selection) - Blockchain Projects

FullStack dApp - DockChain

This FullStack dApp aims at helping boat owners to register for a Docking slip or Mooring buoy thanks to the blockchain.
Boat owners can make a request for a mooring slot at a marina using the DockChain dApp. In case a mooring slot is available then the registered boat will be added to the moored list. Otherwise the boat will be added to the wait list. At anytime the moored list and wait list are accessible.

This FullStack dApp includes a backend part related to the Smart Contract development of the project based on Solidity and a frontend / UI part based on React that allows to interact with the deployed Smart Contract (on Goerli or Hardhat local blockchain).

Main tools and framework used include Hardhat Development Environment, Alchemy API, React, Bulma

Decentralized Crowdfunding platform - Smart Contract

This basic Solidity smart contract facilitates crowdfunding for various projects. It allows project owners to create projects by providing a project title, description, and a specific amount required for participation. Users can participate in projects by funding them with the specified participation amount. The contract keeps track of the total funding amount for each project and the contributions made by individual users. Users can retrieve their contributions by specifying the project ID and their Ethereum address. The project owner can withdraw the total funding amount once the project is completed. The contract also includes fallback and receive functions for receiving Ether, and a getBalance function to retrieve the contract's balance.

#Solidity, #SmartContracts, #Ethereum, #Crowdfunding, #Remix

Decentralized Escrow account - dApp

This dApp is a simple application developed as part of the Alchemy University Dev. Bootcamp Solidity module. It allows a Depositor, which can be an individual or an entity, to make a deposit into an Escrow account and define a Beneficiary and an Arbitror. The deposited amount is securely stored on the blockchain. To unlock or transfer the funds to the Beneficiary, the Arbiter, who acts as an intermediary or trusted third party, can approve the transaction using its wallet. The dApp is implemented using Hardhat and runs on the local blockchain provided by Hardhat. The front-end interface is built using React.

#Solidity, #Hardhat, #Metamask

Voting with NFT - Smart Contract

The MyVotingToken contract is a Solidity smart contract that utilizes the OpenZeppelin library to create a secure voting system based on Non-Fungible Tokens (NFTs). It extends the ERC721 standard and incorporates the draft-ERC721Votes extension from OpenZeppelin. NFT owners can vote for proposals based on their NFT ownership, while the admin can add proposals and display the winning proposal. This contract leverages the OpenZeppelin library's robust functionalities to ensure the integrity and reliability of the voting process.

Best360ies - REST API

This App is the Backend side of the Best360ies Project. The Best360ies-api is an API that serves 360 panoramic images and users database. User can add their favourite 360 panoramic images into their account. It is a server-side application built with Node.js and Express framework. The API sends responses with information about different 360 panoramic images from the database MongoDB to the client. The connection between the server and the database is implemented using mongoose. As a user you need to create a profile and log in. To ensure data security, an authentication and authorization process is established using JWT Token. Only encrypted paswords (hashed) are stored in the database providing extra protection.
